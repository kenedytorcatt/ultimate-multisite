# t450 Brief: Attract More Customers — Positioning Plan

## Session Origin
Issue #450 raised by superdav42: "Make a plan to change our positioning to appeal to more WordPress users or however we can get more customers."

## What
Produce a concrete, actionable positioning and growth plan for Ultimate Multisite. The deliverable is a written plan (this document + the plan doc) that identifies current positioning gaps and specifies what to change, in what order, and why.

## Why
The plugin has strong technical fundamentals (active releases, 186 stars, 73 forks, growing feature set) but low download numbers relative to its capabilities. The most downloaded release (v2.3.4) had 977 downloads — a spike likely driven by the WP Ultimo community discovering the fork. Subsequent releases average 100–270 downloads each. The plugin is not reaching the broader WordPress audience that could benefit from it.

## How
1. Audit current positioning (README, readme.txt, WordPress.org listing copy, GitHub topics)
2. Identify the target audience gap — who is being missed and why
3. Produce a prioritised action plan with specific copy changes, channel actions, and product surface changes

## Acceptance Criteria
- [ ] Current positioning gaps identified with evidence
- [ ] Target audience segments defined with rationale
- [ ] Concrete action plan with ≥5 specific, implementable actions
- [ ] Actions prioritised by impact vs effort
- [ ] Plan committed and linked to issue #450

## Context
- Plugin is a community-maintained fork of WP Ultimo (paid product, now free/open-source)
- Current GitHub topics: `multisite-network`, `wordpress`, `wordpress-plugin` — generic, low-signal
- readme.txt description targets WaaS/SaaS builders — a narrow, technical audience
- WordPress.org is the primary discovery channel for most WordPress users
- The plugin requires WordPress Multisite — a genuine constraint that limits the addressable market but also defines a specific, underserved niche
